### 코인과 토큰의 차이
수요일 토론시간에 이야기나눈 내용과 자체적으로 알아본 내용을 정리했다.

코인 : 비트코인, 이더리움, 솔라나등 자체적인 블록체인 네트워크와 Ledger(~=메인넷)를 갖고 있으며, 네트워크 유지 혹은 사용에 대한 댓가로 사용되는 암호화폐
토큰 : 상기한 코인생태계 위에서 스마트 컨트랙트로 동작하는 암호화폐, 예를 들면.. ERC-20 표준규칙이 있어, 이더리움 스마트컨트랙트 위에서 해당 규격에 맞는 토큰을 누구나 발급할 수 있다.(이더리움만 있다면) '코인'이 블록체인 네트워크를 유지하기 위해 존재한다면, 토큰은 그 블록체인 위에서 부가가치를 창출하기 위해 사용된다고 할수 있다. (실물자산 토큰화, 스테이블 코인 등)

### 스테이블코인은 그럼 토큰..? 코인..?

USDT, USDC와 같은 스테이블 코인들은, 이더리움(ERC-20),트론(TRC-20) 등의 네트워크 위에서 발행되는 토큰들이다. 
달러와 1:1 가치를 유지(페깅)하기위해  발행사(써클, 테더) 등에서는 아래와 같은 방법을 쓴다. 
```
USDC의 가치는 발행사인 써클(Circle)이 보증하는 상환권에서 나옵니다.
• 발행 (Minting): 사용자가 써클에 100달러를 입금하면, 써클은 이더리움과 같은 블록체인 네트워크상에서 100개의 USDC 토큰을 생성하여 사용자 지갑으로 보냅니다.
• 상환 (Redeeming): 사용자가 써클에 100개의 USDC를 돌려주면, 써클은 해당 토큰을 '소각(Burn)'하여 영구히 삭제하고, 보관 중이던 실물 100달러를 사용자의 은행 계좌로 송금합니다.
이 과정이 언제나 가능하기 때문에 시장에서는 "USDC 1개 = 1달러"라는 신뢰가 형성됩니다.
```
*** (번외) 써클의 이더리움 주소 : 0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48

### 금융권에서의 활용 가능성

'코인'자체는 금융권, 특히 우리 은행업에서는 크게 활용의 가치가 없는 것으로 보인다. 사용하게 된다면, 코인 위에서 사용할 '토큰'을 이용하기 위해 유지될텐데..(효용이 상대적으로 떨어짐)
if 신한은행 자체 메인넷 : 외부와 소통이 어려움
else if 퍼블릭 메인넷 : 가스비, 장애가능성, 예측안됨, 공공장부에 고객 거래 기록됨
else if 은행권 공동 메인넷 : 금융결제원과 같은 역할을 하게됨. <- 제일 좋아보인다!!

블록체인을 활용하게 되면 '토큰'을 활용하는 방법이 많아 보인다.
1. 토큰 증권
2. 실물자산토큰화(RWA)
3. 예금 토큰, CBDC 
4. 주식 정산 시스템 등..
각 활용방법에 대한 설명은 정보의 나열이 될것 같으니 생략한다. 요지는 자산의 토큰화이며, 블록체인을 활용하게 되면서, 각 자산의 상태에 대한 검증과 보증에 대한 비용과 공수가 획기적으로 줄어들게 된다는 점이다.
